# 🚀 Configuración del Frontend: Vite + React + Tailwind v4 + shadcn/ui

Este documento resume los pasos necesarios para inicializar un entorno de desarrollo moderno y profesional. El objetivo es dejar lista la infraestructura ("scaffolding") antes de comenzar con la programación de la interfaz.

## 🛠️ Requisitos Previos
- Node.js instalado (versión 18 o superior).
- Un gestor de paquetes como `npm`.

---

## 📦 1. Inicialización del Proyecto
Crea la base del proyecto con la plantilla de TypeScript para asegurar un código más robusto.

```bash
# Crear proyecto con Vite
npm create vite@latest frontend -- --template react-ts

# Entrar a la carpeta e instalar dependencias base
cd frontend
npm install
```

---

## 📦 2. Instalación de Dependencias
Instalamos las librerías de utilidad, el motor de estilos y el plugin de integración para Vite.

```bash
# Librerías de la aplicación
npm install react-router-dom axios react-hook-form

# Herramientas de diseño (Tailwind v4 y tipos de Node)
npm install -D @types/node tailwindcss @tailwindcss/vite
```

---

## ⚙️ 3. Configuración del Entorno

### A. Vite (`vite.config.ts`)
Configura el plugin de Tailwind y los alias de ruta para usar el símbolo `@`.

```typescript
import { defineConfig } from "vite"
import react from "@vitejs/plugin-react"
import tailwindcss from "@tailwindcss/vite"
import path from "path"

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
})
```

### B. TypeScript (`tsconfig.app.json`)
Añade soporte para que el editor de código reconozca los alias de carpetas.

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

### C. Estilos Globales (`src/index.css`)
Sustituye todo el contenido del archivo por la importación de Tailwind v4:

```css
@import "tailwindcss";
```

---

## 🎨 4. Inicialización de shadcn/ui
Ejecuta el asistente para configurar la librería de componentes.

```bash
npx shadcn@latest init
```

**Configuraciones recomendadas durante el proceso:**
- **Style/Preset:** Nova (o New York).
- **Base Color:** Zinc o Slate.
- **Global CSS:** `src/index.css`
- **Import Aliases:** `@/components` y `@/lib/utils`

---

## ✅ Finalidad
Con estos pasos completados, el proyecto cuenta con:
1. **Infraestructura lista:** No habrá errores de rutas ni de compilación de estilos.

3. **Escalabilidad:** Estructura preparada para manejar rutas, formularios y peticiones API desde el primer día.
"""
